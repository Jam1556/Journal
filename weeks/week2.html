SQL

In SQL and databases generally, I covered the basics like records, keys, data types and tables. A record is a row of information, normally personal information is stored here like first name, last name, address etc. Then you have the primary keys that define the records and tell you what they actually are, makes it easier to find specific info. Every record and primary key that you type in must be some kind of data type, whether it be a CHAR, INT, DATE or a boolean. It's up to you what you want it to be but most of the time it's going to be straight forward. For example: "FirstName" is going to be a CHAR, "PhoneNumber" is an INT, "D.O.B" will be DATE and "Employed" with true or false value applied would be a boolean data type. Finally you have the table that brings it all together, it's what is made up of all the records, primary keys, fields and data types that you stored.
